# EXW Master Library: Instructional Design Standards
**Version:** 3.0 (The Guided Learning Integration)

## 🎭 The Google AI Guided Learning Pedagogy
All EXW modules (0-16) must utilize the **Guided Learning Trifecta** to ensure high-friction, human-in-the-loop clinical auditing. Every module must deploy at least one of these roles in its lab or discussion phase:

1.  **The Socratic Professor:** Used for cross-examination and critical thinking. The AI must never give answers; it must only ask probing questions based on the SSoT.
2.  **The Mastery Tutor:** Used for professional analogy and workforce-ready explanations. It bridges complex clinical theory with real-world application.
3.  **The Quiz Master:** Used for low-stakes formative assessment and page-reference verification against the SSoT.

## 🛠️ Design & Enforcement Rules
- **Prompt Chaining:** Multi-step sequential prompting to ensure clinical safety and human-in-the-loop auditing.
- **Assignment Type:** Gemini Socratic Simulation (Checkride).
- **Caveat:** All Guided Learning blocks must explicitly state alignment with AWC AI for All principles.
- **Guided Learning:** Integrated Subject Mastery prompt with Placeholder logic [X, Y, Z, ZZ].
- **Design Rule:** All Role-Plays must include a 'Hidden Trap' safety violation for the student to detect.
- **Scoping:** Each module must declare its SSoT authority in the header to prevent cross-contamination.
- **Tool Building:** Students learn to build custom Gemini assistants (AdaptiveFit) for clinical task automation.
- **Strategy:** Meta-Prompting (using AI to critique its own instructions) is a core requirement for V5.37.
- **Rule:** Explicitly justify the use of Gemini for all clinical simulation tasks to address enterprise filter limitations.
- **Course Cadence:** Standard modules run Tuesday to Monday. Module 16 is the exception: Starts Monday 12:00 am, ends Friday 11:59 pm (Last day of semester).

🏗️ The High-Friction Philosophy

In the AWC Exercise Science program, we do not use Generative AI as a "shortcut." We use it as a Clinical Simulator. By introducing "High-Friction" steps—such as mandatory SSoT uploads and manual logic auditing—we ensure that the student remains the Pilot in Command.

🔄 The Mission Loop (Pattern / Rule / Solve)

Every clinical audit in this course follows the Mission Loop. This framework prevents "lazy" AI usage by forcing the student to verify the machine's logic against professional standards.

1. Pattern (The Intake)

Identify the specific client variables (e.g., T-score, Age, Medications) that define the clinical situation.

2. Rule (The SSoT)

Cite the Single Source of Truth. What does the PAGA 2nd Ed or ACSM 12th Ed say about this specific pattern?

3. Solve (The Prescription)

Synthesize the Pattern and Rule to create a safe, effective exercise intervention.

🛡️ The P-T-C-F Prompting Framework

To communicate effectively with the EXW AI Tool Box, students use the P-T-C-F method to ensure high-fidelity outputs:

  • Persona: (e.g., "Act as a Socratic Professor...")
  • Task: (e.g., "Audit this exercise for spinal flexion safety...")
  • Context: (e.g., "Based on the ACSM 12th Ed guidelines...")
  • Format: (e.g., "Provide the output in a bulleted list.")

🔒 HIPAA & FERPA Compliance

The architecture is built on a Privacy-First foundation. Students are strictly prohibited from uploading any sensitive identifiers (PHI) into public AI models. We use Standardized Patients (e.g., Mrs. Miller) to practice clinical logic without compromising real-world data safety.
